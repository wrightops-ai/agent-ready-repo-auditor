# $750 Agent-Ready Repository Audit

Status: **fixed public scope; opening a request is not a contract or payment obligation**

This service reviews the public operational evidence for one GitHub repository
at one immutable default-branch revision. It is designed for an authorized
maintainer, repository owner, or requester who wants a human-reviewed answer
about whether coding agents can find usable setup, instruction, verification,
contribution, and risky-action boundaries.

## Price, clock, and work gate

- Fixed price: **$750 USD**.
- Delivery: within **three business days** after all required public inputs are
  complete and PayPal Business reports the payment as settled.
- Labor cap: four hours of human review and handoff work.
- Payment: PayPal Goods & Services only, shared privately after WrightOps
  confirms the exact written scope.
- Work does not begin on an authorization, pending payment, hold, invoice,
  checkout visit, or unpaid request.

## Included

- One evidence-linked JSON audit naming the immutable revision.
- One evidence-linked Markdown audit naming the same revision.
- A human-reviewed summary with up to five repository-specific priorities.
- A 30-minute recorded or live handoff.
- Public source links for every positive automated finding.
- Manual review of every positive finding and stated limitation before
  delivery.

## Required public inputs

- One canonical public `https://github.com/owner/repository` URL.
- Confirmation that the requester is authorized to commission the review.
- The coding-agent workflow or operational pain to prioritize.
- A recorded or live handoff preference.

Do not submit credentials, private repository information, customer data,
personal data, contact details, payment information, private files, or
production access through a public GitHub issue.

## Acceptance criteria

- Both delivery formats name the same immutable revision SHA.
- Every positive automated finding links to public GitHub evidence.
- The human-reviewed summary contains no more than five prioritized,
  repository-specific next steps.
- The limitations state that the delivery is not a vulnerability, security,
  privacy, legal, or compliance assessment.
- The report contains no private or customer-supplied data.

## Excluded

- Vulnerability research, security testing, penetration testing, or security,
  privacy, legal, and compliance assessment.
- Repository cloning or execution of repository code.
- Private repositories, credentials, private systems, production access, or
  customer data.
- Code changes, remediation, deployment, infrastructure changes, or ongoing
  monitoring.
- Guaranteed scores, savings, reliability, acceptance, or business outcomes.

the repository, immutable revision, deliverables, exclusions, timing,
acceptance criteria, and handoff format in writing before requesting payment.

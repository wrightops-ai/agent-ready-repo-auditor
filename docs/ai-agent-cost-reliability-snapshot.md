# $495 AI Agent Cost & Reliability Snapshot

Status: **fixed prompt-free scope; opening a request is not a contract or payment obligation**

This service summarizes the supplied normalized operating evidence for one AI-agent workflow. It is designed for an authorized operator who wants an aggregate, inspectable view of cost coverage, completion, first-pass reliability, retries, latency, and evidence gaps without sending prompts, responses, tool payloads, or direct identifiers.

## Price, clock, and work gate

- Fixed price: **$495 USD**.
- Delivery: within **three business days** after all accepted inputs are complete and PayPal Business reports the payment as settled.
- Input limit: one workflow, up to 20 pseudonymous task IDs and 50 normalized attempts.
- Payment: PayPal Goods & Services only, arranged privately after WrightOps confirms the exact written scope.
- Work does not begin on an inquiry, authorization, pending payment, hold, checkout visit, or unpaid request.

## Included

- One deterministic aggregate JSON report naming the normalized input SHA-256.
- One matching human-readable Markdown scorecard.
- A human-reviewed summary with up to five evidence-linked priorities.
- A 30-minute recorded or live readout.
- One factual correction round limited to the accepted evidence and scope.

## Required inputs

- JSONL or CSV records matching the published [`agent-cost-reliability.run.v1` prompt-free contract](ai-agent-cost-reliability-run-contract.md).
- Exactly one non-identifying workflow label.
- Pseudonymous task and run IDs.
- A supplied evidence basis for every outcome and cost field represented as known.
- The one operating decision the scorecard should prioritize.

Do not post run records to the public GitHub request. After written scope confirmation, the accountable owner arranges a private transfer path for the accepted normalized file. Do not supply prompts, responses, tool arguments or results, credentials, customer data, personal data, direct identifiers, private-system details, or production access.

## Acceptance criteria

- Both delivery formats identify the same normalized input SHA-256.
- No run ID, task ID, prompt, response, tool argument, result, credential, or direct identifier appears in the aggregate delivery.
- Missing primary evidence remains explicitly unavailable rather than being treated as zero.
- The reviewed scorecard contains no more than five priorities tied to supplied aggregates.
- Any break-even section is labeled as a user-supplied scenario, never as achieved savings, revenue, or profit.
- Limitations clearly distinguish observation from causal attribution and provider-billed evidence from billing reconciliation.

## Excluded

- Raw prompts, responses, tool arguments or results, credentials, production access, direct identifiers, customer data, or personal data.
- More than one workflow, 20 pseudonymous task IDs, or 50 normalized attempts.
- Data extraction, instrumentation, normalization, implementation, custom evaluation harnesses, or ongoing monitoring.
- Causal attribution, billing reconciliation, certification, or guaranteed savings, reliability, revenue, profit, or business outcomes.
- Security, vulnerability, privacy, legal, compliance, accounting, regulated-work, or professional-advice assessments.
